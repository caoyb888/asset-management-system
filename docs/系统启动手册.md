# 资产管理系统 — 本地开发环境启动手册

> 适用环境：Linux 开发机，Docker 已安装

---

## 一、依赖服务启动

系统共依赖三个基础服务，均以 Docker 容器方式运行。**必须按顺序全部启动后，再启动后端服务。**

### 1. MySQL

```bash
docker start asset-mysql-tmp
```

- 端口：`3306`
- 数据库：`asset_db`
- 账号：`root / root123`

验证：

```bash
docker ps | grep mysql
```

### 2. Redis

```bash
docker start asset-redis
```

- 端口：`6379`，无密码

验证：

```bash
docker ps | grep redis
```

### 3. Nacos（注册 & 配置中心）

```bash
docker start asset-nacos
```

- 端口：`8848`
- Nacos 启动较慢，需等待约 15 秒

验证（返回 `OK` 表示就绪）：

```bash
curl http://127.0.0.1:8848/nacos/v1/console/health/readiness
```

## 二、后端服务启动

### 构建（有代码变更时执行）

```bash
cd /home/testuser/asset-management-system
mvn clean package -pl asset-base -am -DskipTests -q
mvn clean package -pl asset-investment -am -DskipTests -q
```

### 启动 asset-base（基础数据管理）

```bash
nohup java -jar asset-base/target/asset-base-1.0.0-SNAPSHOT.jar \
  --spring.profiles.active=dev \
  > /tmp/asset-base.log 2>&1 &
```

- 端口：`8001`
- Profile：`dev`
- 日志文件：`/tmp/asset-base.log`

### 验证 asset-base

```bash
# 查看启动日志（看到 Started BaseApplication 即成功）
grep "Started\|ERROR" /tmp/asset-base.log | tail -5

# 或直接请求接口（返回 401 说明服务正常，未登录）
curl -s http://127.0.0.1:8001/base/projects?pageNum=1&pageSize=1
```

### 启动 asset-investment（招商管理）

```bash
nohup java -jar asset-investment/target/asset-investment-1.0.0-SNAPSHOT.jar \
  --spring.profiles.active=dev \
  > /tmp/asset-investment.log 2>&1 &
```

- 端口：`8002`
- Profile：`dev`
- 日志文件：`/tmp/asset-investment.log`

### 验证 asset-investment

```bash
# 查看启动日志（看到 Started InvestmentApplication 即成功）
grep "Started\|ERROR" /tmp/asset-investment.log | tail -5
```

## 三、前端启动

```bash
cd /home/testuser/asset-management-system/asset-ui
npm run dev
```

- 默认端口：`3100`
- Vite 代理：`/api/*` → `http://localhost:8001`

## 四、一键启动脚本（快捷方式）

```bash
# 启动所有依赖服务
docker start asset-mysql-tmp asset-redis asset-nacos

# 等待 Nacos 就绪
sleep 15 && curl -s http://127.0.0.1:8848/nacos/v1/console/health/readiness

# 启动后端
nohup java -jar /home/testuser/asset-management-system/asset-base/target/asset-base-1.0.0-SNAPSHOT.jar \
  --spring.profiles.active=dev \
  > /tmp/asset-base.log 2>&1 &

nohup java -jar /home/testuser/asset-management-system/asset-investment/target/asset-investment-1.0.0-SNAPSHOT.jar \
  --spring.profiles.active=dev \
  > /tmp/asset-investment.log 2>&1 &

echo "后端启动中，日志：tail -f /tmp/asset-base.log 或 /tmp/asset-investment.log"
```

## 五、常见启动失败原因

| 错误信息 | 原因 | 解决方法 |
|---|---|---|
| `Connection refused` (3306) | MySQL 未启动 | `docker start asset-mysql-tmp` |
| `Unable to connect to Redis server` | Redis 未启动 | `docker start asset-redis` |
| `Client not connected, current status:STARTING` | Nacos 未启动或未就绪 | `docker start asset-nacos`，等待 15 秒后重试 |
| `Unknown column 'xxx' in 'field list'` | 数据库表结构与实体不一致 | 对照 `sql/init.sql` 检查表字段 |
