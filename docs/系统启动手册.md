# 资产管理系统 — 本地开发环境启动手册

> 适用环境：Linux 开发机，Docker 已安装

---

## 一、依赖服务启动

系统共依赖三个基础服务，均以 Docker 容器方式运行。**必须按顺序全部启动后，再启动后端服务。**

### 1. MySQL

```bash
docker start asset-mysql-tmp
```

- 端口：`3306`
- 数据库：`asset_db`（基础/招商/营运/财务模块）、`asset_report`（报表管理模块）
- 账号：`root / root123`

验证：

```bash
docker ps | grep mysql
```

### 2. Redis

```bash
docker start asset-redis
```

- 端口：`6379`，无密码

验证：

```bash
docker ps | grep redis
```

### 3. Nacos（注册 & 配置中心）

```bash
docker start asset-nacos
```

- 端口：`8848`
- Nacos 启动较慢，需等待约 15 秒

验证（返回 `OK` 表示就绪）：

```bash
curl http://127.0.0.1:8848/nacos/v1/console/health/readiness
```

## 二、后端服务启动

### 构建（有代码变更时执行）

```bash
cd /home/testuser/asset-management-system
mvn clean package -pl asset-base -am -DskipTests -q
mvn clean package -pl asset-investment -am -DskipTests -q
mvn clean package -pl asset-operation -am -DskipTests -q
mvn clean package -pl asset-finance -am -DskipTests -q
mvn clean package -pl asset-report -am -DskipTests -q
mvn clean package -pl asset-system -am -DskipTests -q
mvn clean package -pl asset-workflow -am -DskipTests -q
```

> **提示**：也可一次性构建全部模块：`mvn clean package -DskipTests -q`

```bash
nohup java -jar asset-base/target/asset-base-1.0.0-SNAPSHOT.jar \

  > /tmp/asset-base.log 2>&1 &
```

### 启动 asset-workflow（工作流管理）

> **前置条件**：工作流表已初始化（执行 `sql/workflow_init.sql`，创建 `wf_process_definition`、`wf_process_instance`、`wf_approval_record` 三张表及 8 条流程定义种子数据）

```bash
nohup java -jar asset-workflow/target/asset-workflow-1.0.0-SNAPSHOT.jar \
  --spring.profiles.active=dev \
  > /tmp/asset-workflow.log 2>&1 &
```

- 端口：`8010`
- Profile：`dev`
- 日志文件：`/tmp/asset-workflow.log`
- 说明：基于 Flowable 引擎，提供统一审批提交/通过/驳回/撤回/催办、流程定义管理、审批记录查询
- 审批引擎模式：由 `approval.engine` 配置决定，`mock`（默认，直接通过）或 `flowable`（真实引擎）
- BPMN 流程文件：`asset-workflow/src/main/resources/processes/` 目录下 8 个流程定义
- Flowable 首次启动会自动建表（`flowable.database-schema-update: true`），需要约 30 秒

### 验证 asset-workflow

```bash
# 查看启动日志（看到 Started WorkflowApplication 即成功）
grep "Started\|ERROR" /tmp/asset-workflow.log | tail -5

# 验证 API（返回 401 说明服务正常，未登录）
curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:8010/wf/definitions/page?pageNum=1&pageSize=1
```

> **注意**：业务模块（招商/营运/财务）默认使用 Mock 审批模式，无需启动 asset-workflow 即可正常运行。
> 若需要真实审批流程，需在各业务模块的 `application-dev.yml` 中配置 `approval.engine: flowable`，并确保 asset-workflow 已启动。

## 三、前端启动

```bash
cd /home/testuser/asset-management-system/asset-ui
npm run dev
```

- 默认端口：`3100`
- Vite 代理规则（按匹配优先级）：
  - `/api/auth/*`、`/api/sys/*` → `http://localhost:8006`（系统管理）
  - `/api/wf/*` → `http://localhost:8010`（工作流）
  - `/api/fin/*` → `http://localhost:8004`（财务）
  - `/api/opr/*` → `http://localhost:8003`（营运）
  - `/api/inv/*` → `http://localhost:8002`（招商）
  - `/api/rpt/*` → `http://localhost:8005`（报表）
  - `/api/*`（兜底）→ `http://localhost:8001`（基础数据）

## 四、一键启动脚本（快捷方式）

```bash
# 启动所有依赖服务
docker start asset-mysql-tmp asset-redis asset-nacos

# 等待 Nacos 就绪
sleep 15 && curl -s http://127.0.0.1:8848/nacos/v1/console/health/readiness

BASE=/home/testuser/asset-management-system

# 启动后端（7个服务）
nohup java -jar $BASE/asset-base/target/asset-base-1.0.0-SNAPSHOT.jar \
  --spring.profiles.active=dev > /tmp/asset-base.log 2>&1 &

nohup java -jar $BASE/asset-investment/target/asset-investment-1.0.0-SNAPSHOT.jar \
  --spring.profiles.active=dev > /tmp/asset-investment.log 2>&1 &

nohup java -jar $BASE/asset-operation/target/asset-operation-1.0.0-SNAPSHOT.jar \
  --spring.profiles.active=dev > /tmp/asset-operation.log 2>&1 &

nohup java -jar $BASE/asset-finance/target/asset-finance-1.0.0-SNAPSHOT.jar \
  --spring.profiles.active=dev > /tmp/asset-finance.log 2>&1 &

nohup java -jar $BASE/asset-report/target/asset-report-1.0.0-SNAPSHOT.jar \
  --spring.profiles.active=dev > /tmp/asset-report.log 2>&1 &

nohup java -jar $BASE/asset-system/target/asset-system-1.0.0-SNAPSHOT.jar \
  --spring.profiles.active=dev > /tmp/asset-system.log 2>&1 &

nohup java -jar $BASE/asset-workflow/target/asset-workflow-1.0.0-SNAPSHOT.jar \
  --spring.profiles.active=dev > /tmp/asset-workflow.log 2>&1 &

echo "后端启动中，查看日志："

echo "  tail -f /tmp/asset-operation.log"
echo "  tail -f /tmp/asset-finance.log"

echo "  tail -f /tmp/asset-system.log"
echo "  tail -f /tmp/asset-workflow.log"
```

## 五、常见启动失败原因

| 错误信息 | 原因 | 解决方法 |
|---|---|---|
| `Connection refused` (3306) | MySQL 未启动 | `docker start asset-mysql-tmp` |
| `Unable to connect to Redis server` | Redis 未启动 | `docker start asset-redis` |
| `Client not connected, current status:STARTING` | Nacos 未启动或未就绪 | `docker start asset-nacos`，等待 15 秒后重试 |
| `Unknown column 'xxx' in 'field list'` | 数据库表结构与实体不一致 | 对照 `sql/init.sql` / `sql/finance_v2_init.sql` 检查表字段 |
| `Table 'asset_db.fin_xxx' doesn't exist` | 财务模块建表 SQL 未执行 | 执行 `sql/finance_v2_init.sql` 初始化财务表 |
| `Table 'asset_db.opr_xxx' doesn't exist` | 营运模块建表 SQL 未执行 | 执行 `sql/operation_init.sql` 初始化营运表 |
| `Table 'asset_report.rpt_xxx' doesn't exist` | 报表模块建表 SQL 未执行 | 依次执行 `sql/report_init.sql` → `sql/report_v2_favorite.sql` → `sql/report_v3_schedule.sql` |
| `Unknown column 'report_code' in rpt_user_favorite` | v2 迁移脚本未执行 | 执行 `sql/report_v2_favorite.sql` |
| `Unknown column 'report_code' in rpt_schedule_task` | v3 迁移脚本未执行 | 执行 `sql/report_v3_schedule.sql` |
| `Table 'asset_db.wf_xxx' doesn't exist` | 工作流模块建表 SQL 未执行 | 执行 `sql/workflow_init.sql` 初始化工作流表 |
| `Table 'asset_db.sys_xxx' doesn't exist` | 系统管理模块建表 SQL 未执行 | 依次执行 `sql/system_init.sql`、`sql/system_patch_v2.sql` |
| Flowable 启动慢（首次 30s+） | Flowable 首次启动需自动建表（约 40 张表） | 属正常现象，耐心等待日志输出 `Started WorkflowApplication` |
| `approval.engine` 未配置 | 业务模块默认 Mock 模式 | 若需真实审批：设置 `approval.engine: flowable` 并确保 asset-workflow 已启动 |
| `MP_OPTLOCK_VERSION_ORIGINAL not found` | MyBatis-Plus 乐观锁拦截器未注册（旧版配置） | 确认 `MybatisPlusConfig` 已添加 `OptimisticLockerInnerInterceptor` |

## 七、数据库初始化脚本执行顺序

首次部署时需按以下顺序执行 `sql/` 目录下的初始化脚本：

```bash
# 1. 基础数据（20张表 + admin种子数据）
mysql -u root -proot123 < sql/init.sql

# 2. 招商管理（19张表）
mysql -u root -proot123 < sql/investment_init.sql

# 3. 营运管理（14张表）
mysql -u root -proot123 < sql/operation_init.sql

# 4. 财务管理（13张表，注意用 v2 版本）
mysql -u root -proot123 < sql/finance_v2_init.sql

# 5. 系统管理
mysql -u root -proot123 < sql/system_init.sql
mysql -u root -proot123 < sql/system_patch_v2.sql

# 6. 报表管理（独立库 asset_report，按序执行）
mysql -u root -proot123 < sql/report_init.sql
mysql -u root -proot123 < sql/report_v2_favorite.sql
mysql -u root -proot123 < sql/report_v3_schedule.sql

# 7. 工作流（3张表 + 8条流程定义种子数据）
mysql -u root -proot123 < sql/workflow_init.sql

# 8. 补丁（招商模块租金分解快照）
mysql -u root -proot123 < sql/alter_task7.sql
```

> **注意**：Flowable 引擎在 asset-workflow 首次启动时会自动在 `asset_db` 中创建约 40 张 `ACT_*` / `FLW_*` 表，无需手动执行。
