# 资产管理系统 — 部署手册

> 适用场景：在新服务器上从零拉取代码并部署完整后端服务。
> 当前已完成模块：**基础数据管理**、**招商管理**、**营运管理**、**财务管理**（应收/收款/核销/保证金/预收款/凭证）、**报表管理**（资产/招商/营运/财务四类报表，含收藏与定时推送）

## 二、安装 Docker

### Ubuntu / Debian

```bash
# 卸载旧版本（如有）
sudo apt-get remove -y docker docker-engine docker.io containerd runc

# 安装依赖
sudo apt-get update
sudo apt-get install -y ca-certificates curl gnupg lsb-release

# 添加 Docker 官方 GPG 密钥
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
  | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# 添加 apt 仓库
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" \
  | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装 Docker Engine + Compose Plugin
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io \
  docker-buildx-plugin docker-compose-plugin

# 启动并设置开机自启
sudo systemctl enable --now docker

# 将当前用户加入 docker 组（避免每次 sudo）
sudo usermod -aG docker $USER
newgrp docker    # 立即生效，或重新登录

# 验证
docker version
docker compose version
```

### CentOS 7 / RHEL

```bash
sudo yum remove -y docker docker-client docker-client-latest \
  docker-common docker-latest docker-latest-logrotate docker-logrotate docker-engine

sudo yum install -y yum-utils
sudo yum-config-manager --add-repo \
  https://download.docker.com/linux/centos/docker-ce.repo

sudo yum install -y docker-ce docker-ce-cli containerd.io \
  docker-buildx-plugin docker-compose-plugin

sudo systemctl enable --now docker
sudo usermod -aG docker $USER
newgrp docker
```

## 七、启动基础设施（Docker）

本项目使用三个基础服务：**MySQL 8.0**、**Redis 7**、**Nacos 2.4.3**。

### 7.1 创建 Docker 网络

```bash
docker network create asset-network
```

### 7.2 启动 MySQL

```bash
docker run -d \
  --name asset-mysql \
  --network asset-network \
  -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD=root123 \
  -e MYSQL_DATABASE=asset_db \
  -e TZ=Asia/Shanghai \
  -v asset-mysql-data:/var/lib/mysql \
  --restart unless-stopped \
  mysql:8.0 \
  --character-set-server=utf8mb4 \
  --collation-server=utf8mb4_unicode_ci \
  --default-time-zone=+08:00 \
  --lower-case-table-names=1 \
  --default-authentication-plugin=caching_sha2_password

# 等待 MySQL 就绪（约 20 秒）
until docker exec asset-mysql mysqladmin ping -h localhost --silent 2>/dev/null; do
  echo "等待 MySQL 启动..."; sleep 3
done
echo "MySQL 已就绪"

# 创建报表管理专用数据库
docker exec asset-mysql mysql -uroot -proot123 \
  -e "CREATE DATABASE IF NOT EXISTS asset_report DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
echo "asset_report 数据库已创建"
```

### 7.3 初始化数据库

按顺序依次执行以下 SQL 文件，每次执行均需等待上一条完成。

```bash
# 1. 系统基础表（sys_* / biz_* / inv_* 等）
docker cp sql/init.sql asset-mysql:/tmp/init.sql
docker exec asset-mysql bash -c \
  'mysql --default-character-set=utf8mb4 -uroot -proot123 asset_db < /tmp/init.sql'
echo "基础表初始化完成"

# 2. 营运管理表（opr_* 共 14 张）
docker cp sql/operation_init.sql asset-mysql:/tmp/operation_init.sql
docker exec asset-mysql bash -c \
  'mysql --default-character-set=utf8mb4 -uroot -proot123 asset_db < /tmp/operation_init.sql'
echo "营运表初始化完成"

# 3. 财务管理表（fin_* 共 13 张，使用 V2 版本）
docker cp sql/finance_v2_init.sql asset-mysql:/tmp/finance_v2_init.sql
docker exec asset-mysql bash -c \
  'mysql --default-character-set=utf8mb4 -uroot -proot123 asset_db < /tmp/finance_v2_init.sql'
echo "财务表初始化完成"

# 4. 报表管理表（rpt_* 共 12 张，分三步执行，顺序不可调换）
docker cp sql/report_init.sql asset-mysql:/tmp/report_init.sql
docker exec asset-mysql bash -c \
  'mysql --default-character-set=utf8mb4 -uroot -proot123 < /tmp/report_init.sql'
echo "报表基础表初始化完成（12张表 + 44条配置数据）"

docker cp sql/report_v2_favorite.sql asset-mysql:/tmp/report_v2_favorite.sql
docker exec asset-mysql bash -c \
  'mysql --default-character-set=utf8mb4 -uroot -proot123 < /tmp/report_v2_favorite.sql'
echo "报表收藏表迁移完成（v2）"

docker cp sql/report_v3_schedule.sql asset-mysql:/tmp/report_v3_schedule.sql
docker exec asset-mysql bash -c \
  'mysql --default-character-set=utf8mb4 -uroot -proot123 < /tmp/report_v3_schedule.sql'
echo "报表定时任务表迁移完成（v3）"
```

> **注意**：
> - `finance_v2_init.sql` 会先 DROP 再 CREATE 所有 `fin_*` 表，执行时请确保无生产数据，或提前备份。
> - `operation_init.sql` 使用 `CREATE TABLE IF NOT EXISTS`，可幂等重复执行。
> - `report_init.sql` 使用 `DROP TABLE IF EXISTS`，不可在有报表数据的库上重复执行。v2/v3 脚本为单向迁移，仅执行一次。

验证所有表已创建：

```bash
# 验证 asset_db（基础/招商/营运/财务表）
docker exec asset-mysql mysql -uroot -proot123 asset_db \
  -e "SELECT TABLE_NAME FROM information_schema.TABLES \
      WHERE TABLE_SCHEMA='asset_db' ORDER BY TABLE_NAME;"

# 验证 asset_report（报表表，预期 12 张）
docker exec asset-mysql mysql -uroot -proot123 asset_report \
  -e "SHOW TABLES; SELECT COUNT(*) AS config_count FROM rpt_config WHERE is_deleted=0;"
```

预期：`asset_db` 中 `fin_*`（13 张）、`opr_*`（14 张）及基础表共约 50 张以上；`asset_report` 中 12 张 `rpt_*` 表，`rpt_config` 含 44 条记录。

### 7.4 启动 Redis

```bash
docker run -d \
  --name asset-redis \
  --network asset-network \
  -p 6379:6379 \
  -v asset-redis-data:/data \
  --restart unless-stopped \
  redis:7-alpine \
  redis-server --appendonly yes --maxmemory 256mb --maxmemory-policy allkeys-lru
```

### 7.5 启动 Nacos

```bash
docker run -d \
  --name asset-nacos \
  --network asset-network \
  -p 8848:8848 \
  -p 9848:9848 \
  -e MODE=standalone \
  -e SPRING_DATASOURCE_PLATFORM=mysql \
  -e MYSQL_SERVICE_HOST=asset-mysql \
  -e MYSQL_SERVICE_PORT=3306 \
  -e MYSQL_SERVICE_DB_NAME=nacos_config \
  -e MYSQL_SERVICE_USER=root \
  -e MYSQL_SERVICE_PASSWORD=root123 \
  -e JVM_XMS=256m \
  -e JVM_XMX=256m \
  --restart unless-stopped \
  nacos/nacos-server:v2.4.3

# 等待 Nacos 启动（约 30 秒）
until curl -s http://localhost:8848/nacos/actuator/health | grep -q UP 2>/dev/null; do
  echo "等待 Nacos 启动..."; sleep 5
done
echo "Nacos 已就绪，访问 http://<服务器IP>:8848/nacos  账号: nacos/nacos"
```

### 7.6 验证所有服务

```bash
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"
```

预期输出：

```
NAMES           STATUS          PORTS
asset-nacos     Up X seconds    0.0.0.0:8848->8848/tcp, 0.0.0.0:9848->9848/tcp
asset-redis     Up X seconds    0.0.0.0:6379->6379/tcp
asset-mysql     Up X seconds    0.0.0.0:3306->3306/tcp
```

## 八、编译后端

```bash
cd asset-management-system

# 第一步：安装公共模块到本地 Maven 仓库（顺序不可调换）
mvn clean install -pl asset-common,asset-common-mybatis,asset-common-log,asset-common-security \
  -DskipTests -q

# 第二步：打包各业务模块
mvn clean package -pl asset-base       -DskipTests -q
mvn clean package -pl asset-investment -DskipTests -q
mvn clean package -pl asset-operation  -DskipTests -q
mvn clean package -pl asset-finance    -DskipTests -q
mvn clean package -pl asset-report     -DskipTests -q

# 验证 JAR 均已生成
ls -lh asset-base/target/asset-base-*.jar \
       asset-investment/target/asset-investment-*.jar \
       asset-operation/target/asset-operation-*.jar \
       asset-finance/target/asset-finance-*.jar \
       asset-report/target/asset-report-*.jar
```

> **注意**：每次修改公共模块（`asset-common-*`）后，必须先对该模块执行 `mvn clean install`，
> 再重新打包依赖它的业务模块，否则 Maven 会使用本地仓库中的旧版本。

如需一次性运行单元测试后再打包（CI 场景）：

```bash
# 财务模块含 17 个单元测试，需 H2 内存库，无需外部依赖
mvn clean package -pl asset-finance -am
```

### 10.2 生产构建

```bash
cd asset-ui
npm install
npm run build
# 构建产物在 asset-ui/dist/

# 用 nginx 提供静态文件（示例）
docker run -d \
  --name asset-ui \
  -p 80:80 \
  -v $(pwd)/dist:/usr/share/nginx/html:ro \
  -v $(pwd)/nginx.conf:/etc/nginx/conf.d/default.conf:ro \
  nginx:alpine
```

`nginx.conf` 示例（按模块分路由反代到各后端）：

```nginx
server {
    listen 80;
    root /usr/share/nginx/html;
    index index.html;

    # 前端路由（history 模式）
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 报表管理（/api/rpt/* → 8005，必须在 /api/ 之前声明，否则被基础服务拦截）
    location /api/rpt/ {
        proxy_pass http://<服务器IP>:8005/rpt/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 招商管理（/api/inv/* → 8002）
    location /api/inv/ {
        proxy_pass http://<服务器IP>:8002/inv/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 营运管理（/api/opr/* → 8003）
    location /api/opr/ {
        proxy_pass http://<服务器IP>:8003/opr/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 财务管理（/api/fin/* → 8004）
    location /api/fin/ {
        proxy_pass http://<服务器IP>:8004/fin/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 文件服务（/api/file/* → 8012）
    location /api/file/ {
        proxy_pass http://<服务器IP>:8012/file/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 基础数据管理（/api/* → 8001，兜底规则，必须放最后）
    location /api/ {
        proxy_pass http://<服务器IP>:8001/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 十二、常见问题

### Q1：MySQL 启动后无法连接

```bash
# 查看容器日志
docker logs asset-mysql --tail 50

# 常见原因：端口 3306 被占用
ss -tlnp | grep 3306
```

### Q2：后端启动报 SM2 解密失败

前端登录密码经 SM2 加密传输，**不影响正常使用**。此为 WARN 级别日志，系统会自动回退明文模式（仅开发环境）。

### Q3：前端显示乱码

向数据库写入中文数据时，必须指定 utf8mb4 字符集。使用 `docker cp` + 文件方式执行 SQL，禁止在 `docker exec -e` 中直接传递含中文的 SQL 字符串。

```bash
# 正确方式
docker cp your_data.sql asset-mysql:/tmp/data.sql
docker exec asset-mysql bash -c \
  'mysql --default-character-set=utf8mb4 -uroot -proot123 asset_db < /tmp/data.sql'
```

### Q4：Maven 编译子模块后主模块未更新

```bash
# 修改 asset-common-security 后必须先 install
mvn clean install -pl asset-common-security -DskipTests
# 再打包主模块
mvn clean package -pl asset-base -DskipTests
```

### Q5：Nacos 启动慢或连接 MySQL 失败

Nacos 依赖 MySQL 的 `nacos_config` 库，该库由 Nacos 容器自动创建。确保 MySQL 完全就绪后再启动 Nacos：

```bash
# 检查 nacos_config 库是否创建成功
docker exec asset-mysql mysql -uroot -proot123 \
  -e "SHOW DATABASES LIKE 'nacos_config';"
```

### Q6：财务模块启动报 fin_xxx 表不存在

```bash
# 执行财务 V2 初始化脚本（幂等：先 DROP 后 CREATE）
docker cp sql/finance_v2_init.sql asset-mysql:/tmp/finance_v2_init.sql
docker exec asset-mysql bash -c \
  'mysql --default-character-set=utf8mb4 -uroot -proot123 asset_db < /tmp/finance_v2_init.sql'

# 验证 13 张财务表
docker exec asset-mysql mysql -uroot -proot123 asset_db \
  -e "SHOW TABLES LIKE 'fin_%';"
```


### Q10：报表接口返回 500，错误含 `report_code` 字段不存在

v2 或 v3 迁移脚本未执行，参考 Q9 执行对应脚本。

### Q7：营运模块启动报 opr_xxx 表不存在

```bash
docker cp sql/operation_init.sql asset-mysql:/tmp/operation_init.sql
docker exec asset-mysql bash -c \
  'mysql --default-character-set=utf8mb4 -uroot -proot123 asset_db < /tmp/operation_init.sql'

# 验证 14 张营运表
docker exec asset-mysql mysql -uroot -proot123 asset_db \
  -e "SHOW TABLES LIKE 'opr_%';"
```

### Q8：财务模块核销/保证金操作报乐观锁参数错误

错误信息：`Parameter 'MP_OPTLOCK_VERSION_ORIGINAL' not found`

原因：`asset-common-mybatis` 中 `MybatisPlusConfig` 未注册 `OptimisticLockerInnerInterceptor`。
已在 v1.0 中修复，拉取最新代码重新编译即可：

```bash
mvn clean install -pl asset-common-mybatis -DskipTests
mvn clean package -pl asset-finance -DskipTests
```
