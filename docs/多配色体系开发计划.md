# 多配色体系开发计划

## 背景

当前系统仅有一套蓝色主题（#2E75B6），且大量颜色硬编码在 50+ 个 Vue 文件中。计划增加 3-4 种配色体系，支持用户在界面上切换，提升系统视觉个性化能力。

## 技术方案

### 核心思路：CSS 自定义属性 + `data-theme` 属性切换

- 在 `<html>` 上设置 `data-theme="teal|indigo|dark"` 属性
- 通过 `:root` 和 `:root[data-theme="xxx"]` 选择器定义不同主题的 CSS 变量
- Pinia store 管理当前主题 + localStorage 持久化
- `main.ts` 中在 Vue 挂载前立即读取 localStorage 设置 `data-theme`（避免闪烁）

### 为什么不用 SCSS 变量切换
SCSS 变量在编译时确定，无法运行时切换。必须改用 CSS 自定义属性（`var(--xxx)`）。现有 SCSS 变量保留为别名（`$primary-color: var(--app-color-primary)`），避免改动所有 `<style scoped>` 块。

## 任务分解

### 阶段一：基础设施（P0，必做）

#### T1. 创建主题定义文件 `src/styles/themes.scss`
- 定义 CSS 自定义属性命名规范：`--app-{类别}-{名称}`
- `:root` 定义默认蓝色主题的所有变量
- `[data-theme="teal"]`、`[data-theme="indigo"]`、`[data-theme="dark"]` 分别定义覆盖值
- 包含：主色、侧边栏、头部、内容区、文字、卡片、边框等约 20 个变量
- 同时覆盖 Element Plus 变量（`--el-color-primary` 等）
- **文件：** 新建 `src/styles/themes.scss`

#### T2. 重构 `variables.scss` 为 CSS 变量别名
- 将 SCSS 硬编码值改为引用 CSS 自定义属性
- 删除 `:root` 块（移入 themes.scss）
- 保留 `$sidebar-width` 等不需要主题切换的布局变量
- **文件：** `src/styles/variables.scss`

#### T3. 更新样式入口
- `index.scss` 中 import themes.scss
- **文件：** `src/styles/index.scss`

#### T4. Store 增加主题状态
- `AppState` 新增 `theme: string`（默认 `''` 表示经典蓝）
- 新增 `setTheme(theme)` action：设置 `document.documentElement.dataset.theme`，写入 localStorage
- **文件：** `src/store/modules/app.ts`

#### T5. 应用启动时恢复主题
- `main.ts` 中在 `app.mount()` 前读取 `localStorage.getItem('asset_theme')`，立即应用到 `<html>`
- **文件：** `src/main.ts`

#### T6. 主题切换 UI + 侧边栏迁移
- `BasicLayout.vue` 头部右侧（用户下拉旁）添加主题切换器（4 个色块圆点或下拉）
- 移除 `el-menu` 的硬编码 `background-color="#001529"` 和 `text-color` props
- 改为通过 CSS 覆盖 el-menu 内部变量（`--el-menu-bg-color` 等），使其跟随主题
- 侧边栏 `.layout-sidebar` 的 `background-color` 改用 `var(--app-sidebar-bg)`
- **文件：** `src/layouts/BasicLayout.vue`

#### T7. 创建 `useThemeColors` composable
- 提供 JS 可用的响应式主题颜色（用于 ECharts、内联 style 等无法用 CSS 变量的场景）
- 内置 4 套 ECharts 图表配色方案
- watch `appStore.theme` 变化时更新
- **文件：** 新建 `src/composables/useThemeColors.ts`

### 阶段二：高频页面适配（P1，推荐）

#### T8. 工作台首页适配
- `dashboard/index.vue` 的模块卡片颜色、统计卡片颜色改用 `useThemeColors()` 或 CSS 变量

#### T11. 其余报表页 ECharts 适配

- 其余暗色背景保持不变
- **文件：** `src/views/login/index.vue`

## 关键决策说明

| 决策 | 理由 |
|------|------|
| 登录页不强制跟随主题 | 独立品牌设计，每次会话只看一次，改造成本大收益低 |
| 语义色（成功/警告/危险）不随主题变 | 这是 UI 规范而非品牌色，跨主题保持一致减少用户困惑 |
| SCSS 变量保留为 CSS 变量别名 | 避免修改所有 `<style scoped>` 块，渐进式迁移 |
| el-menu 从 props 改 CSS | 内联 props 优先级高于 CSS，必须移除才能被主题变量控制 |
| ECharts 用 JS 配色方案而非读 CSS | ECharts 不识别 CSS 变量，JS 配色方案更可控、性能更好 |

## 验证方式

1. 切换每种主题后检查：
   - 侧边栏背景色、选中态正确
   - 头部样式正确
   - 内容区背景色正确
   - Element Plus 组件（按钮、链接、分页等）主色跟随
   - ECharts 图表配色跟随
2. 刷新页面后主题保持（localStorage 持久化）
3. 暗色模式下文字对比度、卡片边框可见性检查
4. 登录页不受主题影响（或仅按钮色微调）
