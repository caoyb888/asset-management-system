# 用户自定义扩展字段方案

> 版本：v1.0 | 日期：2026-03-18 | 状态：待评审

---

## 1. 需求背景

用户测试阶段反馈：部分模块的字段不够用，需要增加业务字段。为避免每次字段变更都修改后端代码、重新部署，需要提供一种**用户可自主配置**的扩展字段机制——管理员在系统管理界面增删字段，前端即时展示，无需开发介入。

### 适用范围

以下模块的核心业务实体均可支持扩展字段：

| 模块 | 业务表 | 说明 |
|------|--------|------|
| 基础数据 | biz_project, biz_shop, biz_brand, biz_merchant | 项目、商铺、品牌、商家 |
| 招商管理 | inv_intention, inv_lease_contract | 意向协议、租赁合同 |
| 营运管理 | opr_contract_ledger, opr_contract_change | 合同台账、合同变更 |
| 财务管理 | fin_receivable, fin_receipt | 应收、收款 |

> 可按需逐步扩展到其他表，方案本身不限制表数量。

---

## 2. 方案选型

| 方案 | 原理 | 优点 | 缺点 | 结论 |
|------|------|------|------|------|
| **A. EAV 模型** | 实体-属性-值三元组表存储 | 灵活度最高，无需 DDL | 查询性能差，JOIN 爆炸，报表困难 | 不推荐 |
| **B. JSON 字段** | 业务表加 `ext_fields JSON` 列 | 实现简单，MySQL 8 原生 JSON 查询，与现有表结构兼容 | JSON 索引能力有限 | **推荐** |
| **C. 动态 DDL** | 运行时 ALTER TABLE 加列 | 查询性能最好 | 风险高，DDL 锁表，权限控制复杂 | 不推荐 |

**最终选择：方案 B — JSON 扩展字段 + 字段元数据配置**

---

## 3. 架构设计

### 3.1 整体架构

```
┌──────────────────────┐
│   系统管理 - 字段配置  │  管理员在此配置各模块的扩展字段
│   /sys/ext-fields     │  （字段名、类型、选项、是否必填等）
└──────────┬───────────┘
           │ 元数据
           ▼
┌──────────────────────┐     ┌──────────────────────┐
│  sys_ext_field_def   │     │   业务表              │
│  字段元数据定义表      │     │   ext_fields JSON 列  │
│  (存字段结构)         │     │   (存字段值)          │
└──────────┬───────────┘     └──────────┬───────────┘
           │                            │
           ▼                            ▼
┌─────────────────────────────────────────────────────┐
│                    前端动态渲染                        │
│  ExtFieldRenderer.vue  →  表单动态生成                │
│  ExtFieldColumns.vue   →  列表动态列                  │
│  useExtFields.ts       →  元数据加载 + 缓存           │
└─────────────────────────────────────────────────────┘
```

### 3.2 数据流

```
1. 管理员配置字段
   系统管理页面 → POST /sys/ext-fields → 写入 sys_ext_field_def

2. 业务人员使用
   打开业务页面 → GET /sys/ext-fields?moduleCode=xxx → 获取字段元数据
   → 前端动态渲染表单项和列表列
   → 用户填写扩展字段值 → 随业务数据一起保存到 ext_fields JSON 列

3. 数据查询
   业务列表 → 后端返回含 extFields 的业务对象 → 前端根据元数据渲染对应列
```

## 4. 数据库设计

### 4.1 新增表：字段元数据定义 `sys_ext_field_def`

```sql
CREATE TABLE sys_ext_field_def (
    id           BIGINT       PRIMARY KEY AUTO_INCREMENT COMMENT '主键',
    module_code  VARCHAR(32)  NOT NULL COMMENT '模块编码（如 project/shop/contract/merchant）',
    field_key    VARCHAR(64)  NOT NULL COMMENT '字段标识（英文，用作 JSON key）',
    field_label  VARCHAR(64)  NOT NULL COMMENT '字段显示名称（中文）',
    field_type   VARCHAR(16)  NOT NULL COMMENT '字段类型：text/number/date/select/radio/checkbox/textarea',
    options_json JSON         NULL     COMMENT '下拉/单选/多选的选项列表 [{\"label\":\"xxx\",\"value\":\"xxx\"}]',
    required     TINYINT(1)   NOT NULL DEFAULT 0 COMMENT '是否必填：0-否 1-是',
    sort_order   INT          NOT NULL DEFAULT 0 COMMENT '排序序号（升序）',
    show_in_list TINYINT(1)   NOT NULL DEFAULT 0 COMMENT '是否在列表页展示：0-否 1-是',
    show_in_form TINYINT(1)   NOT NULL DEFAULT 1 COMMENT '是否在表单页展示：0-否 1-是',
    default_val  VARCHAR(255) NULL     COMMENT '默认值',
    placeholder  VARCHAR(128) NULL     COMMENT '输入提示文本',
    max_length   INT          NULL     COMMENT '最大长度（text/textarea 适用）',
    min_val      DECIMAL(18,4) NULL    COMMENT '最小值（number 适用）',
    max_val      DECIMAL(18,4) NULL    COMMENT '最大值（number 适用）',
    is_deleted   TINYINT(1)   NOT NULL DEFAULT 0 COMMENT '逻辑删除：0-正常 1-删除',
    created_by   VARCHAR(64)  NULL     COMMENT '创建人',
    created_at   DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_by   VARCHAR(64)  NULL     COMMENT '更新人',
    updated_at   DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    UNIQUE KEY uk_module_field (module_code, field_key)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户自定义扩展字段定义';
```

| module_code | 对应业务表 | 说明 |

| brand | biz_brand | 品牌 |
| merchant | biz_merchant | 商家 |
| intention | inv_intention | 意向协议 |
| contract | inv_lease_contract | 租赁合同 |
| ledger | opr_contract_ledger | 合同台账 |
| change | opr_contract_change | 合同变更 |
| receivable | fin_receivable | 应收 |
| receipt | fin_receipt | 收款 |

**field_type 取值说明：**

| field_type | 渲染组件 | 说明 |
|------------|----------|------|

| textarea | el-input type="textarea" | 多行文本 |

## 5. 后端设计

### 5.1 实体类

**字段定义实体 — `SysExtFieldDef.java`**（asset-system 模块）

```java
@Data
@TableName("sys_ext_field_def")
public class SysExtFieldDef extends BaseEntity {
    private Long id;
    private String moduleCode;
    private String fieldKey;
    private String fieldLabel;
    private String fieldType;

    @TableField(typeHandler = JacksonTypeHandler.class)
    private List<Map<String, String>> optionsJson;

    private Boolean required;
    private Integer sortOrder;
    private Boolean showInList;
    private Boolean showInForm;
    private String defaultVal;
    private String placeholder;
    private Integer maxLength;
    private BigDecimal minVal;
    private BigDecimal maxVal;
}
```

**业务实体增加扩展字段**（以 BizProject 为例）

```java
// 在现有 Entity 中加一个字段即可
@TableField(typeHandler = JacksonTypeHandler.class)
private Map<String, Object> extFields;
```

> 所有需要支持扩展字段的 Entity 均加此字段，改动量极小。

### 5.3 缓存策略

字段定义变更频率低但查询频率高（每次打开业务页面都要获取），使用 Redis 缓存：

```
缓存 Key:   sys:ext:field:{moduleCode}
缓存 TTL:   60 分钟
失效时机:   新增/修改/删除字段定义时主动清除
```

### 5.4 业务接口无需改动

现有业务接口（如 `POST /base/projects`）的请求体和响应体中，`extFields` 作为一个普通的 JSON 对象字段，MyBatis-Plus 的 `JacksonTypeHandler` 自动处理序列化/反序列化，**无需修改任何业务 Service 和 Controller 代码**。

## 6. 前端设计

### 6.1 公共 Composable — `useExtFields.ts`

```typescript
// src/composables/useExtFields.ts
import { ref, onMounted } from 'vue'
import { getExtFieldDefs } from '@/api/sys/extField'

export interface ExtFieldDef {
  id: number
  moduleCode: string
  fieldKey: string
  fieldLabel: string
  fieldType: 'text' | 'textarea' | 'number' | 'date' | 'select' | 'radio' | 'checkbox'
  optionsJson?: { label: string; value: string }[]
  required: boolean
  sortOrder: number
  showInList: boolean
  showInForm: boolean
  defaultVal?: string
  placeholder?: string
  maxLength?: number
  minVal?: number
  maxVal?: number
}

// 内存缓存，同一页面多组件共享
const cache = new Map<string, ExtFieldDef[]>()

export function useExtFields(moduleCode: string) {
  const fieldDefs = ref<ExtFieldDef[]>([])
  const loading = ref(false)

  const formFields = computed(() => fieldDefs.value.filter(f => f.showInForm))
  const listFields = computed(() => fieldDefs.value.filter(f => f.showInList))

  onMounted(async () => {
    if (cache.has(moduleCode)) {
      fieldDefs.value = cache.get(moduleCode)!
      return
    }
    loading.value = true
    const data = await getExtFieldDefs(moduleCode)
    fieldDefs.value = data
    cache.set(moduleCode, data)
    loading.value = false
  })

  // 生成默认值对象（新增表单时调用）
  function getDefaults(): Record<string, any> {
    const defaults: Record<string, any> = {}
    for (const f of fieldDefs.value) {
      if (f.defaultVal != null) {
        defaults[f.fieldKey] = f.fieldType === 'number'
          ? Number(f.defaultVal)
          : f.defaultVal
      }
    }
    return defaults
  }

  return { fieldDefs, formFields, listFields, loading, getDefaults }
}
```

### 6.2 通用表单组件 — `ExtFieldRenderer.vue`

```vue
<template>
  <template v-for="field in formFields" :key="field.fieldKey">
    <el-form-item
      :label="field.fieldLabel"
      :prop="'extFields.' + field.fieldKey"
      :rules="field.required ? [{ required: true, message: `请输入${field.fieldLabel}` }] : []"
    >
      <!-- 单行文本 -->
      <el-input
        v-if="field.fieldType === 'text'"
        v-model="model[field.fieldKey]"
        :placeholder="field.placeholder"
        :maxlength="field.maxLength"
      />

      <!-- 多行文本 -->
      <el-input
        v-else-if="field.fieldType === 'textarea'"
        v-model="model[field.fieldKey]"
        type="textarea"
        :rows="3"
        :placeholder="field.placeholder"
        :maxlength="field.maxLength"
      />

      <!-- 数字 -->
      <el-input-number
        v-else-if="field.fieldType === 'number'"
        v-model="model[field.fieldKey]"
        :min="field.minVal"
        :max="field.maxVal"
        :placeholder="field.placeholder"
      />

      <!-- 日期 -->
      <el-date-picker
        v-else-if="field.fieldType === 'date'"
        v-model="model[field.fieldKey]"
        type="date"
        value-format="YYYY-MM-DD"
        :placeholder="field.placeholder"
      />

      <!-- 下拉选择 -->
      <el-select
        v-else-if="field.fieldType === 'select'"
        v-model="model[field.fieldKey]"
        :placeholder="field.placeholder"
      >
        <el-option
          v-for="opt in field.optionsJson"
          :key="opt.value"
          :label="opt.label"
          :value="opt.value"
        />
      </el-select>

      <!-- 单选 -->
      <el-radio-group
        v-else-if="field.fieldType === 'radio'"
        v-model="model[field.fieldKey]"
      >
        <el-radio
          v-for="opt in field.optionsJson"
          :key="opt.value"
          :value="opt.value"
        >{{ opt.label }}</el-radio>
      </el-radio-group>

      <!-- 多选 -->
      <el-checkbox-group
        v-else-if="field.fieldType === 'checkbox'"
        v-model="model[field.fieldKey]"
      >
        <el-checkbox
          v-for="opt in field.optionsJson"
          :key="opt.value"
          :value="opt.value"
        >{{ opt.label }}</el-checkbox>
      </el-checkbox-group>
    </el-form-item>
  </template>
</template>

<script setup lang="ts">
import { useExtFields } from '@/composables/useExtFields'

const props = defineProps<{
  moduleCode: string
  modelValue: Record<string, any>
}>()

const emit = defineEmits(['update:modelValue'])

const model = computed({
  get: () => props.modelValue || {},
  set: (val) => emit('update:modelValue', val)
})

const { formFields } = useExtFields(props.moduleCode)
</script>
```

### 6.3 通用列表列组件 — `ExtFieldColumns.vue`

```vue
<template>
  <el-table-column
    v-for="field in listFields"
    :key="field.fieldKey"
    :label="field.fieldLabel"
    :prop="'extFields.' + field.fieldKey"
    :min-width="120"
  >
    <template #default="{ row }">
      {{ formatValue(field, row.extFields?.[field.fieldKey]) }}
    </template>
  </el-table-column>
</template>

<script setup lang="ts">
import { useExtFields, type ExtFieldDef } from '@/composables/useExtFields'

const props = defineProps<{ moduleCode: string }>()
const { listFields } = useExtFields(props.moduleCode)

function formatValue(field: ExtFieldDef, val: any): string {
  if (val == null) return '-'
  if (field.fieldType === 'checkbox' && Array.isArray(val)) {
    // 多选：将 value 转为 label 展示
    const labelMap = new Map(field.optionsJson?.map(o => [o.value, o.label]))
    return val.map(v => labelMap.get(v) || v).join('、')
  }
  if (field.fieldType === 'select' || field.fieldType === 'radio') {
    // 单选/下拉：将 value 转为 label
    return field.optionsJson?.find(o => o.value === val)?.label || val
  }
  return String(val)
}
</script>
```

## 8. 风险与约束

| 风险项 | 影响 | 应对措施 |
|--------|------|----------|
| JSON 字段无法建索引 | 扩展字段不适合作为高频查询条件 | 高频查询字段仍应加为正式数据库列 |
| 扩展字段值无数据库约束 | 类型/必填校验依赖前端 | 后端保存时根据元数据做二次校验 |
| 删除字段定义后历史数据残留 | ext_fields 中仍有旧 key | 不影响业务，前端不再渲染即可；可提供数据清理工具 |
| 字段过多影响页面布局 | 表单过长、列表列过多 | 配置页限制单模块最大字段数（建议 ≤ 20） |

## 9. 后续扩展方向

以下能力可在 v1.0 上线后按需迭代：

1. **扩展字段搜索** — 利用 MySQL 8 的 `JSON_EXTRACT` 支持按扩展字段条件筛选
2. **扩展字段导出** — EasyExcel 导出时动态追加扩展字段列
3. **字段分组** — 支持将扩展字段按分组展示（如"财务信息"、"运营信息"）
4. **字段权限** — 结合 RBAC 控制不同角色可见/可编辑的扩展字段
5. **字段模板** — 预设常用字段组合，一键导入到指定模块
6. **审计日志** — 记录扩展字段值的变更历史
