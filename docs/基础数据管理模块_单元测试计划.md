## 二、测试策略

### 2.1 测试类型分工

| 测试类型 | 工具 | 说明 |
|----------|------|------|
| **Service 单元测试** | JUnit 5 + Mockito | Mock Mapper 层，验证纯业务逻辑分支 |
| **Controller 集成测试** | MockMvc + 真实 MySQL | 验证 HTTP 入参校验、事务完整性、数据库写入 |

### 2.2 测试框架版本

```xml
<!-- pom.xml 依赖 -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-test</artifactId>
    <scope>test</scope>
    <!-- 包含: JUnit 5.x, Mockito 5.x, AssertJ, MockMvc -->
</dependency>
<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-inline</artifactId>
    <scope>test</scope>
    <!-- 支持 static 方法 mock（SmCryptoUtil） -->
</dependency>
```

### 2.3 关键约定

- 集成测试使用 `@Transactional`，每个 `@Test` 方法结束后**自动回滚**，保证用例隔离
- 开发环境**永久基础数据**通过 `sql/test-data-dev.sql` 脚本写入，不受事务回滚影响（数据 ID 固定在 90000+ 范围）
- Service 单元测试**不启动 Spring 容器**，执行速度 < 50ms / 用例
- 测试配置文件：`src/test/resources/application-test.yml`（已禁用 Nacos、连接本地 MySQL）

## 六、测试数据维护规范

1. **永久数据**：`sql/test-data-dev.sql` 使用固定 ID（90001~90099），脚本首行加 `SET FOREIGN_KEY_CHECKS=0;`，执行后恢复，幂等可重跑
2. **临时数据**：集成测试中通过 `@BeforeEach` 注入，随 `@Transactional` 自动回滚，不污染永久数据
3. **加密数据**：`biz_merchant.id_card` 字段在 `test-data-dev.sql` 中存储 SM4 密文（加密原文参见 SQL 注释）
4. **禁止操作**：不得直接 `DELETE/TRUNCATE` 永久测试表，若需重置可重跑 `sql/test-data-dev.sql`
