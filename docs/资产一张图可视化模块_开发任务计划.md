# 资产一张图可视化模块 — 开发任务计划

> **项目**：产城公司资产管理系统
> **模块**：资产一张图可视化（Module 5 / asset-gis）
> **技术栈**：Spring Boot 3.x + MyBatis-Plus + Redis + Vue 3 + 高德地图 + ECharts 5
> **端口**：8007 | 控制器前缀：`/gis/*`
> **文档版本**：v1.0 · 2026-03-24

---

## 一、模块概览

资产一张图通过 GIS 地图 + BI 报表实现资产数据的空间化、可视化展示，支持多维度钻取和分级权限控制。数据主要来源于基础数据、招商、营运、财务模块的汇总数据，GIS 层仅存储坐标、快照汇总、趋势数据和看板配置。

**数据库表清单（共 8 张）**

| 分类 | 数据表 |
|------|--------|
| GIS 坐标 | gis_project_location |
| 看板快照 | gis_dashboard_snapshot、gis_resource_type_stat、gis_area_ranking、gis_risk_detail |
| 趋势数据 | gis_trend_data |
| 看板配置 | gis_dashboard_config |
| 运维日志 | gis_snapshot_log |

**功能点总计**：38 个（P0 = 22 个，P1 = 10 个，P2 = 6 个）

**依赖模块**：asset-base（项目/商铺）、asset-investment（合同）、asset-finance（应收/收款）、asset-common-security（JWT + 数据权限）

**预估工期**：3 天

**ETL 快照生成逻辑（00-07 核心）：**

```
全量快照流程：
1. 查询所有项目 → 按省/市分组 → 生成 scope_type=2(省)/3(市)/4(项目) 快照
2. 汇总所有省级数据 → 生成 scope_type=1(全国) 快照
3. 每个快照关联生成：资源类型统计 + 面积排行 + 风险明细
4. 记录 gis_snapshot_log

数据来源 SQL：
- 项目/资源数 → biz_project + biz_shop（COUNT/SUM GROUP BY project_id）
- 面积统计 → biz_shop（SUM rent_area WHERE shop_status）
- 空置率 → 空置面积 / 总面积 * 100
- 应收/已收 → fin_receivable（SUM original_amount / received_amount）
- 合同到期 → inv_lease_contract（WHERE contract_end BETWEEN NOW AND +30天）
```

## 六、技术难点与应对方案

| 难点 | 等级 | 应对方案 |
|------|------|---------|
| GIS 地图与业务数据融合渲染 | 高 | 高德地图 WebGL 渲染万级标记点；服务端 GeoHash 聚合减少前端渲染量；信息弹窗懒加载 |
